# Client

Ученики авторизуются, принимают приглашения в группы
и тренируются по карточкам из доступных наборов.

## Зависимости

### Внутренние

Зависит от `@onlycards/service` и `@onlycards/ui`:
импортирует типы и функции для работы с внешними данными,
а также UI карточек.

### Внешние

В разработке применяются React, Zustand и SCSS.
Не используем роутер.

## Запросы к сервису

### Авторизация

```typescript
import { logClientIn, logClientOut } from '@onlycards/service'
```

### Данные пользователя

```typescript
import { getClientData } from '@onlycards/service'
```

## Логика работы

Авторизационные данные HTTP-only,
так что мы не можем заранее явно проверить их наличие.

При старте приложения вызываем `getClientData`. Если ответ `null`, значит,
пользователь не авторизован и мы показываем страницу авторизации.

> "Страница" — это просто компонент, который мы сейчас показываем.
> Страница никак не влияет на URL. Роутер не используется.

Если данные есть, показываем главную страницу. Здесь можно:

- выйти из профиля;
- увидеть свои ID и ник;
- посмотреть информацию о группах и колодах;
- выбрать любое количество колод и начать тренировку.

При запуске тренировки переходим на страницу тренировки.
Описание тренировки будет позже.

## Архитектура
